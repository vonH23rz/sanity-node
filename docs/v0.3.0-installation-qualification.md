# Sanity Node v0.3.0 — Installation qualification

Status: complete; Docker installation path qualified.

## Purpose

This document records the isolated fresh-install qualification of the
public Sanity Node Docker Compose deployment before the `v0.3.0`
release.

The qualification verifies more than initial startup. It confirms that
a clean installation can bootstrap safely, validate its configuration,
build and start, generate and serve the public dashboard, restart,
survive an invalid scheduled refresh without replacing its last
successful output, recover automatically, and clean up completely.

The qualification does not create or move a release tag.

The qualified source revision was:

  `Fail closed on invalid Docker refresh configuration`.

The isolated qualification host used:

- Docker Engine `29.5.3`, build `d1c06ef`;

The installation was required to prove all of the following:

## Defect discovered and correction

The first qualification attempt exposed a fail-open refresh defect.

The original Docker startup path validated configuration during initial
startup, but scheduled refreshes invoked the generator directly. When
malformed YAML was introduced after startup, the generator interpreted
the invalid configuration as empty configuration, selected reference
mode, enabled reference collectors, and replaced the last successful
public dashboard.

The observed failure signature was:

```text
GENERATOR_RESULT=SUCCESS
OUTPUT_RESULT=REPLACED
RUNTIME_RESULT=FELL_BACK_TO_REFERENCE
COLLECTOR_RESULT=REFERENCE_COLLECTORS_ENABLED
```

Commit `9954b91` corrected the Docker refresh contract.

Initial startup and every scheduled refresh now use this sequence:

```text
validate configuration
→ run startup preflight
→ run generator
```

If configuration validation or startup preflight fails, the generator
is not executed.

## Invalid-refresh rejection

The corrected fresh installation was tested by introducing malformed
YAML before the next scheduled refresh.

The runtime logged:

```text
ERROR: Could not parse YAML
Configuration validation failed; generator was not run
Keeping the last successfully generated dashboard
```

The qualification confirmed that:

- configuration validation rejected the malformed YAML;
- startup preflight and generation were not entered after validation
  failed;
- the generator invocation count did not increase;
- the dashboard content hash did not change;
- dashboard ownership, mode, and modification metadata did not change;
- HTTP continued serving the last successful dashboard;
- the container remained running and healthy;
- reference collectors were not enabled;
- runtime mode did not fall back to `reference`.

This is the required last-successful-output protection.

## Automatic recovery

The valid configuration was restored without restarting or rebuilding
the container.

At the next scheduled refresh, the runtime logged:

```text
Configuration validation passed.
Running Sanity Node startup preflight
Running Sanity Node generator
Generator finished successfully
```

Generation resumed automatically, the served dashboard advanced
normally, and the container remained healthy.

No manual output restoration was required.

## Qualification result

The corrected public Docker installation path is technically qualified.

A clean user installation can:

- bootstrap safely;
- validate and preflight before generation;
- build and start through Docker Compose;
- become healthy;

a release limitation.
